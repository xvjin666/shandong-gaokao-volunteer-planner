# 开发进度控制文档

## 1. 项目阶段

### 阶段 0：方案与风险框架

状态：已启动。

交付物：

- 完整系统方案。
- 数据源清单。
- 决策依据和反证机制定义。

完成标准：

- 所有推荐必须带证据链。
- 所有结论必须标注数据来源和版本。
- 明确 AI 使用边界。

### 阶段 1：数据底座

目标：

- 下载并归档官方原始文件。
- 建立统一字段结构。
- 支持 CSV/XLSX 导入。
- 对 `.xls` 官方文件保留原始文件并提供转换流程。

关键任务：

- 建立 `data/sources/sdzk_official_sources.json`。
- 实现 `gaokao_decision.importer`。
- 记录导入版本、字段映射和异常行。

质量门槛：

- 每条记录必须有 `source_id`。
- 最低分、最低位次、计划数必须能被校验为数字。
- 官方记录数量与导入数量必须生成差异报告。

当前状态：

- 已实现 SQLite schema、导入批次、导入校验报告。
- 已下载并归档 2023-2025 官方投档表原始 `.xls`。
- 已下载并归档 2023-2025 官方夏季高考文化成绩一分一段表。
- 已实现山东考试院普通类常规批第 1 次志愿投档表转换器。
- 历史本地工作库曾导入官方转换数据和一分一段数据；开源仓库不发布该库。
- 开源默认库路径使用 `data/local/gaokao.sqlite` 或 `data/sample/open_demo.sqlite`。
- 2024、2025 各有 1 条清华大学专业缺最低位次，已作为 warning 保留。

### 阶段 2：推荐内核

目标：

- 实现硬性过滤。
- 实现三年位次加权。
- 实现趋势和稳定性分析。
- 输出可解释推荐项。

质量门槛：

- 单元测试覆盖风险分层、过滤规则、证据链。
- 合成样例可以稳定运行。
- 任何推荐缺少证据时必须降级或报错。

当前状态：

- 已实现硬性过滤、三年位次加权、趋势识别、稳定性评分、证据链、反证条件。
- 已实现 `recommend` 和 `recommend-db`。
- 已实现 `plan` 和 `plan-db`，支持 aggressive、balanced、conservative 三种冲稳保策略。
- 已实现 `rank-plan-db`，支持输入 2026 全省位次，按 2023-2025 官方一分一段表换算同位次等效分，再按等效分上下浮动筛选候选。
- 已实现 `scripts/serve_app.py` 本地五步法网页系统。
- 当前测试：`PYTHONPATH=src python3 -m unittest discover -s tests` 通过。

### 阶段 3：正式数据接入

目标：

- 清洗 2023-2025 官方投档表。
- 接入 2026 招生计划和选科要求。
- 建立人工抽样核验流程。

质量门槛：

- 每年随机抽样不少于 100 条，与官方 Excel 原行对照。
- 对热门学校、热门专业、低位次保底段额外抽样。
- 对新增专业、改名专业、中外合作、校区变化单独标记。

### 阶段 4：Web UI

目标：

- 做本地浏览器操作界面。
- 实现问卷输入、推荐列表、证据抽屉、替代项比较、导出。

质量门槛：

- 非专业用户不需要理解算法也能完成输入。
- 每个风险标签都能展开看依据。
- 用户能手动锁定、删除、替换志愿。
- 最终导出 Excel/PDF。

### 阶段 5：AI 辅助

目标：

- 增加偏好访谈。
- 增加报告生成。
- 增加 AI 二次复核。

质量门槛：

- AI 输出必须引用系统内已有证据。
- AI 不得新增未经证实的数据。
- AI 建议和规则模型冲突时必须标红提示人工判断。

## 2. 风险登记表

| 风险 | 影响 | 控制措施 |
| --- | --- | --- |
| 官方 `.xls` 解析不稳定 | 数据无法直接入库 | 保留原始文件，允许先转换为 CSV/XLSX 后导入 |
| 2026 计划变化大 | 历史位次预测失效 | 推荐项加入计划变化反证条件 |
| 专业备注限制遗漏 | 错报不可填志愿 | 建立备注字段人工复核清单 |
| AI 幻觉 | 误导用户 | AI 只解释和复核，不直接改推荐 |
| 用户偏好表达不清 | 推荐不符合真实意愿 | 使用问卷和排斥项二次确认 |
| 保底不足 | 滑档风险 | 强制生成保底充分性报告 |

## 3. 数据校验清单

每次导入必须检查：

- 文件来源 URL。
- 文件下载时间。
- 文件 hash。
- 表头识别结果。
- 行数。
- 空位次数量。
- 空分数数量。
- 重复院校专业数量。
- 异常低分和异常高位次。
- 字段映射是否需要人工确认。

## 4. 推荐结果校验清单

每次生成方案必须检查：

- 是否所有推荐都满足选科。
- 是否所有推荐都满足学费限制。
- 是否存在用户明确排斥的专业。
- 是否存在用户不接受的学校性质或合作办学。
- 冲稳保比例是否合理。
- 保底志愿是否足够。
- 前 20 个志愿是否真的是用户愿意去的项目。
- 每条推荐是否有证据链和反证条件。
